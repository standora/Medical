# 医共体中心药房系统 — 产品设计规格

## 1. 产品定位

| 维度 | 定义 |
|------|------|
| 产品名称 | 医共体智慧药房协同平台 |
| 产品类型 | 药房+协同混合型纯前端演示平台 |
| 目标客户 | 20万人口以下微型县域医共体 |
| 核心价值 | 零库存托管 + 自动补货 + 中西医一体化 + 智能算法驱动 |
| 部署架构 | 纯前端SPA应用 + Mock数据层，静态部署（Vercel/Netlify/GitHub Pages） |
| 商业模式 | 低门槛SaaS（基础年费 + 按处方量计费） |

### 竞争差异化

| 差异化卖点 | 行业现状 | 我们的优势 |
|-----------|---------|-----------|
| 村卫生室零库存托管 | 7家竞品均未实现 | 全行业首创，村医只开方不碰药 |
| 自动补货算法 | 7家竞品均未实现 | AI驱动需求预测，自动生成补货订单 |
| 中西医一体化药房 | 西药强中药弱 或 中药强西药弱 | V2.0实现中西医全流程深度管理 |
| 智能算法驱动 | 仅图特科技有基础算法 | 智能调剂/预警/补货三引擎 |

### 技术架构（纯前端）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| UI框架 | React 18 + TypeScript | 组件化开发，类型安全 |
| UI组件库 | Ant Design 5 | 企业级组件，开箱即用 |
| 状态管理 | Zustand | 轻量级状态管理，替代Redux |
| 路由 | React Router v6 | SPA路由管理 |
| 图表 | ECharts / @ant-design/charts | 统计分析可视化 |
| Mock数据 | MSW (Mock Service Worker) + 自定义Mock生成器 | 拦截API请求，返回模拟数据 |

| 构建工具 | Vite | 快速开发与构建 |
| 部署 | Vercel / Netlify / 静态文件 | 零服务器部署 |

### 4.3 统一库存管理（含零库存托管 — 杀手功能）

| 功能点 | 说明 | 实现级别 |
|--------|------|---------|
| 三级库存架构 | 总院中心库→分院药库→村卫生室药房三级联动 | 完整 |
| 库存预警 | 上下限预警 + 近效期预警 + 消耗趋势缺货预测 | 进阶 |
| 库存调剂 | 跨机构库存查询 + 智能调剂算法推荐最优路径 | 进阶 |
| 盘点管理 | 全盘/抽盘，自动盘点报告 | 进阶 |
| 村卫生室零库存托管 | 村医只开方不碰药，库存由分院/总院托管，全流程自动化 | 完整（行业首创） |
| 自动补货算法 | 基于历史消耗+季节性+流行病趋势预测需求，自动生成补货订单 | 完整（行业首创） |

**零库存托管核心流程**：
```
村医开方 → 处方上传至分院/总院 → 药房调配 → 配送到村/到户 → 村医确认签收
```

**自动补货算法逻辑（前端TS实现）**：
```
输入：历史消耗数据 + 季节因子 + 流行病预警 + 安全库存天数
算法：加权移动平均 + 季节性分解 + 异常值过滤
输出：补货品种 + 补货数量 + 建议补货时间
触发：库存低于安全阈值 / 手动触发
人工干预：自动生成的补货单需药师确认后方可提交采购
```

**Mock数据**：预生成三级机构库存数据（含预警项）、零库存托管配置、自动补货记录

**页面交互**：
- 库存总览：三级机构树形切换，库存卡片+预警标记
- 零库存托管：村卫生室配置面板，托管状态可视化
- 自动补货：算法结果列表，每条显示置信度+原因，药师确认/拒绝按钮
- 智能调剂：调剂路径可视化（机构间箭头连线），推荐调剂方案

### 4.9 智能算法引擎（前端TS实现）

| 算法 | 输入 | 输出 | 触发条件 |
|------|------|------|---------|
| 自动补货算法 | 历史消耗+季节因子+流行病预警+安全库存天数 | 补货品种+数量+时间 | 库存低于安全阈值/手动触发 |
| 智能调剂算法 | 各机构库存+距离+紧急程度+运输成本 | 最优调剂路径+调剂数量 | 手动触发/缺货预警触发 |
| 智能预警算法 | 消耗趋势+效期+库存水位 | 预警等级+处理建议 | 实时监控 |

**算法实现策略**：
- V1.0采用规则引擎+统计模型（加权移动平均、季节性分解），纯TypeScript前端实现
- V2.0引入机器学习模型（时序预测、异常检测）
- V1.0所有算法输出必须经药师/管理员人工确认后方可执行，不开放自动执行
- V2.0根据客户信任度逐步开放自动执行模式（可配置开关）
